    lit-html

    Awesome Lit ¶

    >精选的lit-html和LitElement资源列表.

    lit-html —用于JavaScript的高效，可表达，可扩展的HTML模板库.

    General resources¶

    • Documentation
    • GitHub repository

    Community¶

    • lit-html Twitter
    • Slack Channel

    Overview¶

    • Lightning-fast templates & Web Components: lit-html & LitElement

    Implementations¶

    按创建日期排序（从最旧的开始）.

    • lit-element -用于创建快速，轻量级Web组件的简单基类.
    • @gluon/gluon -用于构建Web组件和应用程序的轻量级库.
    • @lit-any/core -Partials库，它使用lit-html从较小的模板组成视图和表单.
    • lit-html-element -使用lit-html创建Web组件的基类.
    • fit-html -lit-html，Web组件和Redux的组合.
    • lit-html-brackets -一个lit-html扩展名，使用类似于Angular模板语法的括号语法.
    • @littleq/element-lite -尝试使用lit-html和Polymer的属性mixin.
    • @popeindustries/lit-html-server -将Node.js流上的lit-html模板呈现在服务器上.
    • ullr -使用lit-html通过功能编程构建Web组件.
    • ce-decorators -用于基于装饰器提案和lit-html构建Web组件的TypeScript库.
    • @appnest/lit-translate -轻量级的i18n库，提供lit-html指令.
    • haunted -React的Hooks API，但适用于标准Web组件和hyperHTML或lit-html.
    • fuco -功能组件，如React，但用于Web组件.
    • lit-up -具有lit-html模板的反应式Web应用程序的最小但可扩展的状态模式
    • lit-app -用于前端应用程序，服务器端呈现和静态站点的基于同构lit-html的上下文
    • perlite -一种声明性的方式，可以创建专门针对服务器端应用程序设计的富客户端小部件.

    Renderers¶

    以下库允许将lit-html用作渲染器，同时还提供其他替代方法，例如 HyperHTML 或JSX.

    • @corpuscule/lit-html-renderer -基于lit-html的渲染器 Corpuscule.
    • @helikopter/render-lit-html -基于lit-html的渲染器 Helicopter.
    • @moleculejs/molecule-lit -基于lit-html的渲染器 Molecule.
    • @skatejs/element-lit-html -基于lit-html的渲染器 SkateJS.
    • icomponent-lit -基于lit-html的渲染器 icomponent.

    LitElement Extensions¶

    这些不是lit-html本身的实现，而是官方LitElement基类的社区扩展.

    • @adobe/lit-mobx -将mobx与LitElement一起使用的Mixin和基类.
    • @apollo-elements/lit-apollo -LitElement与Apollo GraphQL的集成.
    • @collaborne/lit-flexbox-literals -提供LitElement CSS文字和从移植的类 @polymer/iron-flex-layout.
    • @morbidick/lit-element-notify -用于LitElement的小型mixin，可通过属性getter获得轻松的更改事件.
    • fluent-localize-mixin -LitElement扩展，提供开箱即用的轻松10n.
    • haunted-lit-element -之间缺少连接 Haunted 和LitElement.
    • lit-robot -与LitElement集成 Robot.
    • lit-element-state -LitElement的简单共享应用程序状态管理.
    • nextbone -通过ES类和装饰器重新构想了与LitElement一起使用的主干网.
    • wc-context -具有LitElement集成的Web组件的上下文实现.
    • lit-element-state-decoupler -用于在LitElement组件外部进行状态处理的实用程序.

    Inspired Solutions¶

    这些库不依赖lit-html，但是受到其某些概念的启发. 他们使用标记为“ html” 模板文字，并利用相同的好处 IDE Plugins 用于语法突出显示.

    • htm -超标标记的标记：使用标准标记模板的JSX替代品，并具有编译器支持.
    • hybrids -用于使用简单实用的API创建Web组件的UI库.
