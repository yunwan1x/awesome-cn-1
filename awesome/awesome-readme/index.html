    Tools¶

    • Common Readme -Node的通用自述文件样式. 包括指南和自述文件生成器.
    • Feedmereadmes -自述文件编辑和项目分析/反馈.
    • Hall-of-fame -帮助在README上显示对回购贡献者的认可. 拥有新的/趋势/杰出贡献者. 每小时更新一次.
    • Make a README -编写自述文件的指南. 包括带有实时Markdown渲染的可编辑模板.
    • README best practices -从以下位置复制粘贴您的README.md的地方

    • Readme.so -一个简单的编辑器，可让您快速添加和自定义项目自述文件所需的所有部分.
    • READMINE -用于软件项目的透彻，清晰和自描述的README文件模板； 复制并根据需要进行编辑.
    • Standard Readme -标准的README样式规范. 也有一个生成器来帮助创建符合规范的自述文件.
    • Zalando's README Template -简单的模板，可帮助您涵盖所有基础知识.

    Creating GIFs¶

    在自述文件中嵌入动画gif可以快速演示您的项目做什么，并引起读者的注意. 以下是一些程序，可以帮助您快速为项目创建gif：

    • Gifox - $ 4.99 -最干净的用户界面，热键和许多高级功能
    • Gifski -免费-比其他颜色更鲜艳，但尺寸仍然较小
    • Giphy Capture -免费-易于上传到giphy.com，用户体验有些烦人.
    • LICEcap -免费-不太直观，更多功能
    • Peek -免费-简单易用的Linux用户.
    • Recordit -免费-简单，干净的用户界面，但会自动上传到 recordit.co
    • ScreenToGif -免费-开源，可自定义的用户界面和易于编辑的GIF，易于上手.
    • terminalizer -免费-记录您的终端并生成动画gif图像或共享网络播放器
    • ttystudio -免费-对于命令行工具，使用终端到gif记录器可以减轻麻烦.
